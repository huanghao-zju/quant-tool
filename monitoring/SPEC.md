# 危机前置指标自动监控系统 — 需求规格（SPEC）

> 本文档是项目的唯一权威需求来源，供 Claude Code 在每次会话中阅读。
> 背景：2026年7月的宏观分析结论——下半年全球系统性金融危机概率约3–5%，若爆发，条件概率最高的起点是日本（"日元贬值失控 → 政策被迫转向 → 套息平仓"两段式序列，约30–35%）。本工具的任务是把该分析中的前置指标变成自动监控 + 分级告警。
> 版本：v1.0 SPEC（2026-07-04）。阈值基于当时市场状态标定，需每季度人工复核。

---

## 0. 设计哲学（最高优先级约束）

1. **状态机而非单指标告警。** 单指标告警必然产生告警疲劳，最终被用户忽略。系统只在**阶段迁移**时推送，平时只发周报摘要。
2. **速度型阈值优先于水平型。** 日元通道的历史教训（2024年8月）：USDJPY 的水平（161）不重要，两周10円的速度才致命。每个时间序列必须同时计算水平条件与变化率条件。
3. **先回测证伪，后上线。** 三段历史窗口回测不通过（见 §7），不得部署。
4. **工具是提醒器，不是决策者。** 决策已写在 §1 的"信号→动作表"中，工具的唯一职责是在正确的时间把正确的表行推送到用户面前。
5. **v1 从简。** 一个周末能跑通的粗糙版本，胜过计划中的完美版本。v1 只做 FRED + yfinance 可覆盖的指标；爬取类全部放 v2（见 §9）。

## 1. 信号→动作表（用户必须在部署前填写，代码将其原样嵌入推送文案）

> ⚠️ 占位：以下动作列由用户本人填写具体标的、比例、执行时限。工具在阶段迁移推送时，必须把对应行的动作文案完整附在告警消息中。

| 阶段迁移 | 动作（用户填写，示例格式） |
|---|---|
| 绿 → 黄 | 例：停止一切高beta加仓；复核持仓X/Y当前仓位 |
| 黄 → 橙 | 例：24小时内将标的X减至总仓位N%以内，不做二次确认 |
| 橙 → 红 | 例：高beta仓位全部执行既定下限；启用现金弹药清单 |
| 任意 → 降级 | 例：恢复常规节奏，记录本次事件复盘 |

动作表存放于 `config/actions.yaml`，与阈值配置分离，便于独立修订。

## 3. 双阈值体系（默认值，全部可在 `config/thresholds.yaml` 中修改）

### 3.1 日元通道（快状态机）

| 指标 | 水平型 | 速度型 |
|---|---|---|
| USDJPY | ≥165（贬值失控确认线）；≤150（Mode A 深度确认） | 单周（5个交易日）下跌 >5円 = 急升预警；两周下跌 >10円 = 2024年8月级别（**注意方向：数值下跌 = 日元升值**） |
| 10Y JGB | >2.3% | 单周 +15bp |
| 30Y JGB | 创发行以来新高 | 单周 +15bp |
| 30Y 拍卖（v2） | 投标倍数 <3.0 | 尾差相对12个月均值显著走阔 |
| CFTC 日元净头寸 | 净空头处于2年分位数 >90%（拥挤确认） | 从极值两周内骤减 >50% = 平仓进行中 |
| 日经225 | — | 单日 <-5%（与纳指同日下跌形成合取 → 传导确认） |

### 3.2 全局信用/宏观（慢变量看板）

| 指标 | 阈值分档 |
|---|---|
| HY OAS | 350bp（预警）/ 400bp（进入 distress，重估全部结论）/ 600bp（Tier 2 领域） |
| VIX | 收盘 >25，或单日跳升 >8点 |
| SOFR − IORB | 利差转正且连续5个交易日走阔（回购压力） |
| SRF 用量 | 非季末/非月末出现异常用量（>历史常态一个量级） |
| 布伦特 | >95美元（地缘再升级 / 日本贸易条件恶化加速） |
| 非农 | 连续两个月环比负增长（若同期美联储仍在加息 = 政策错误窗口，标记为复合信号） |
| Sahm | ≥0.50 |
| OWL/ARES | 5个交易日内 >-10%（AI信用代理触发） |
| BIZD | 相对52周高点回撤 >15% |

## 4. 阶段状态机（日元通道主线）

系统维护一个持久化状态 `state.json`（含当前阶段、进入时间、触发条件快照）。

### 阶段定义与迁移条件

**阶段0 · 绿色（基线）**
初始状态。行为：仅每周一次摘要推送（全部指标现值 + 距阈值距离）。

**阶段0 → 阶段1 · 黄色（Mode B 确认：贬值失控）**
满足以下任一：
- USDJPY ≥165 且连续2个收盘确认；
- 30Y JGB 创发行以来新高 且 10Y >2.3%；
- （v2）30Y 拍卖投标倍数 <3.0 且尾差恶化；
- `events.yaml` 录入"财政包超预期"类事件。

**阶段1 → 阶段2 · 橙色（政策被迫转向：转折点）**
满足以下任一：
- USDJPY 从近30日高点单周急跌 >5円（干预成功或政策转向的市场印记）；
- `events.yaml` 录入"BOJ会议间行动/加速加息信号"或"MOF无预告大规模干预"；
- CFTC 净空头从极值开始骤减（两周 >30%）。

**阶段2 → 阶段3 · 红色（Mode A：套息平仓 + 全球传导）**
满足以下任一：
- USDJPY 两周下跌 >10円 且（VIX 单日 +8 或 日经与纳指同日 <-3%）；
- CFTC 净空头两周骤减 >50%；
- HY OAS 突破 400bp（全局信用确认，任何阶段直接跳红）。

### 跳级与降级规则
- 允许跳级：任一更高阶段条件直接满足时，直接迁移到该阶段（例如绿色状态下 USDJPY 两周急跌10円 → 直接红色）。
- 降级：连续 **10个交易日** 无当前阶段及以上任何条件满足 → 降一级；降级同样推送（附复盘提示）。
- 慢变量看板（§3.2）不参与状态机分级（HY OAS>400 例外），以独立"看板分"形式附在每次推送尾部：满足条件的慢变量个数 n/9。

## 5. 去抖与推送策略

1. **确认规则**：所有日度水平型条件需连续2个收盘确认；速度型条件即时生效（速度本身就是信号）。
2. **推送时机**：仅在（a）阶段迁移、（b）每周一次定期摘要（建议周六早晨北京时间）、（c）慢变量看板分变动 时推送。
3. **推送内容模板**：`[阶段迁移] 绿→黄 | 触发条件：… | 当前读数：… | ➤ 你的既定动作（来自 actions.yaml 原文）：… | 数据时间戳`。
4. **推送渠道**：Server酱 或 企业微信群机器人 webhook（URL 存 GitHub Secrets，环境变量 `PUSH_WEBHOOK_URL`）。推送失败需重试3次并在 Actions 日志留痕。
5. **心跳**：每周摘要即心跳；若任何数据源连续3天拉取失败，单独推送数据源故障告警（防止"沉默的坏掉"）。

## 6. 技术架构

- **语言/运行时**：Python 3.11+。依赖尽量少：`requests`、`pandas`、`yfinance`、`pyyaml`。FRED 用原生 REST（免费 API key，环境变量 `FRED_API_KEY`）而非重型 SDK。
- **调度**：GitHub Actions cron。工作日每日两跑：UTC 22:00（美国收盘后，覆盖 FRED/美股类）与 UTC 07:30（东京收盘后，覆盖 JGB/日经/USDJPY 亚洲时段）。零服务器成本。
- **状态持久化**：`state.json` 提交回仓库（Actions 内 git commit），或用 Actions cache/artifact。必须能在两次运行间保持阶段状态与历史序列缓存。
- **目录结构**：

```
crisis-monitor/
├── CLAUDE.md              # 指向本 SPEC + 编码约定
├── SPEC.md                # 本文档
├── config/
│   ├── thresholds.yaml    # §3 全部阈值
│   ├── actions.yaml       # §1 信号→动作表
│   └── events.yaml        # §2.3-20 人工定性事件
├── src/
│   ├── fetchers/          # 每数据源一个模块，统一返回 pd.Series
│   ├── signals.py         # 水平/速度双条件计算
│   ├── state_machine.py   # §4 状态机
│   ├── notify.py          # 推送
│   └── main.py
├── backtest/
│   ├── run_backtest.py    # §7 三段窗口
│   └── data/              # 回测用历史数据缓存
├── state.json
└── .github/workflows/monitor.yml
```

- **编码约定**：所有阈值与逻辑参数一律来自 yaml，代码中不得出现魔法数字；每个 fetcher 必须处理数据缺失/接口变更并降级（跳过该指标 + 记入故障计数），单一数据源故障不得中断整体运行。

## 7. 回测要求（部署前验收门槛）

状态机必须在以下三段历史数据上重放并满足验收标准（用 FRED/yfinance 历史数据 + 手工补录当期 CFTC 数据；`events.yaml` 按当时公开新闻的实际日期补录）：

| 窗口 | 场景 | 验收标准 |
|---|---|---|
| 2024-06-01 ~ 2024-09-30 | 日元套息平仓（真阳性） | 必须在 **2024-08-02 或之前**到达橙色（即8月5日崩盘前给出提前量），8月5日当日或之前到达红色 |
| 2026-02-01 ~ 2026-04-30 | 伊朗冲击（通道区分） | 油价/VIX 慢变量看板应触发，但日元状态机**不得升至红色**（验证系统能区分冲击通道） |
| 2025-07-01 ~ 2025-12-31 | 平静期（假阳性） | 升至阶段1以上的次数 ≤2；若超过，收紧阈值重跑 |

回测报告输出：每次阶段迁移的日期、触发条件、相对事件锚点的提前/滞后天数。**任何一条不达标，调参重跑，不得带病上线。**

## 8. v1 / v2 范围切分

- **v1（一个周末封顶）**：§2.1 全部 12 个指标 + §2.2 中的 JGB 收益率与 CFTC（若时间不够则降入 v2）；状态机 + 去抖 + 推送 + 2024 窗口回测。
- **v2**：§2.3 全部爬取/解析类；三段完整回测；隐含相关性指数；财报解析。
- **明确不做**：任何自动交易执行；任何付费数据源；任何需要常驻服务器的组件。

## 9. 已知数据源注意事项

- FRED `BAMLH0A0HYM2` 滞后约1个交易日，且该系列自2026年4月起仅保留3年观测（回测取数用 FRED 图形接口的长历史或本地缓存）。
- yfinance 为非官方接口，需容错（重试 + 空值跳过）。
- CFTC Socrata 每周五发布周二数据，速度型计算需按报告日期而非发布日期对齐。
- 财务省 CSV 与 NY Fed API 在国内网络环境可能不稳定，fetcher 需支持代理配置（环境变量 `HTTPS_PROXY`），JGB 数据回退 akshare。
- 本 SPEC 中所有阈值基于 2026-07-04 的市场状态标定（USDJPY≈162、HY OAS≈263bp、政策利率 3.50–3.75%/1.00%），**每季度人工复核一次，市场结构性变化后必须重新标定**。
